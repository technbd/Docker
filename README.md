
## Docker:

`Docker` is an open-source platform used to building, deploying, and run applications inside containers. Containers package an application and its dependencies into a lightweight, portable unit, enabling consistent behavior across different environments.


### Key aspects of Docker:

1. Docker Engine:
    - The Docker Engine is the core component of Docker, responsible for creating, running, and managing containers. It consists of:
        - `Docker Daemon` (`dockerd`): **The background service that manages Docker containers, images, networks, and volumes.**
        - `Docker CLI`: A command-line interface that allows users to interact with Docker via commands like docker run, docker pull, and docker build.

2. Containers:
    - A container is a lightweight, standalone, and executable software package that includes everything needed to run an application (code, runtime, libraries, environment variables, etc.).
    - Containers are isolated from each other and from the host system, but they share the same operating system kernel, making them more efficient than virtual machines (VMs).

3. Images:
    - A Docker image is a read-only template used to create containers.
    - Images contain the application code, dependencies, and system libraries required to run the container.

4. Dockerfile:
    - A `Dockerfile` is a text file that contains instructions to build a Docker image. It automates the process of creating custom Docker images by specifying the base image, dependencies, environment variables, and application code.

5. Volumes:
    - Volumes are a way to persist data generated by Docker containers. Since containers are ephemeral, data stored within them is lost when the container is stopped or removed.
    - Volumes allow data to persist beyond the life cycle of a container and can also be shared between multiple containers.

6. Networking:
    - Docker provides several networking modes to connect containers:
        - `Bridge`: The default networking mode for containers on a standalone Docker host. Containers can communicate with each other through the Docker bridge network.
        - `Host`: The container shares the host's network namespace, which can lead to performance improvements but removes isolation.
        - `None`: No networking for the container.
        - `Overlay`: Used in Docker Swarm mode for cross-host communication.

7. Docker Hub:
    - Docker Hub is a public repository for Docker images. It allows users to share, distribute, and download images. Docker Hub contains a vast collection of official and community-contributed images.
