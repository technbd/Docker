## Docker Logs: 

Docker logs refer to the output streams from running containers, specifically capturing standard output (`STDOUT`) and standard error (`STDERR`) generated by the containerized applications. These logs are crucial for monitoring, debugging, and understanding the behavior of applications within Docker containers.



The `docker logs` command shows the standard output (stdout) and standard error (stderr) streams from a container.

_Basic Syntax:_
```
docker logs <container_id_or_name>

docker logs -f <container_id_or_name>
```



### Logging Drivers:

Docker supports multiple logging backends (drivers):

| Logging Driver | Description |
|----------------|-------------|
| **none** | No logs are available for the container and `docker logs` does not return any output. |
| **local** | Logs are stored in a custom format designed for minimal overhead. |
| **json-file (default)** | The logs are formatted as JSON. This is the default logging driver for Docker. |
| **syslog** | Writes logging messages to the `syslog` facility. The `syslog` daemon must be running on the host machine. |
| **journald** | Writes log messages to `journald`. The `journald` daemon must be running on the host machine. |
| **gelf** | Writes log messages to a Graylog Extended Log Format (GELF) endpoint such as Graylog or Logstash. |
| **fluentd** | Writes log messages to `fluentd` (forward input). The `fluentd` daemon must be running on the host machine. |
| **awslogs** | Writes log messages to Amazon CloudWatch Logs. |
| **splunk** | Writes log messages to `splunk` using the HTTP Event Collector. |
| **etwlogs** | Writes log messages as Event Tracing for Windows (ETW) events. Only available on Windows platforms. |
| **gcplogs** | Writes log messages to Google Cloud Platform (GCP) Logging. |




### Examples:

To sets the **logging driver** for this container to **syslog**, meaning logs will be sent to the **system syslog daemon** (e.g: `/var/log/syslog` or `/var/log/messages` depending on your OS).

```
docker run --name myapp -d --log-driver syslog nginx
```


Or,


Check container logs through **systemd journal** instead of `docker logs` like `journalctl -u docker`, `journalctl -f CONTAINER_NAME=myapp1`, `journalctl CONTAINER_ID=<container_id>`:
```
docker run --name myapp1 -d --log-driver journald nginx
```



### Set Default Logging Driver in Docker Daemon:

_To set default Logging Driver or globally (`/etc/docker/daemon.json`) and when you set the logging driver to `local` like this:_
```
{
  "log-driver": "local"
}
```


```
systemctl restart docker
```




### What `local` does?
- Docker stores container logs in a **custom local format** on the host.
- The format is designed for **minimal disk and performance overhead** compared to json-file.
- Logs are **persisted on the host**, so they survive container restarts.
- You can still access them using the docker logs command.
- By default, `local` logs are stored under the container’s directory on the host:
```
/var/lib/docker/containers/<container-id>/local-logs/container.log
```
