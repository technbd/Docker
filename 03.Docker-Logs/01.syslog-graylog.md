## Graylog as a Syslog Server:

Graylog is a powerful centralized log management system that aggregates logs from servers, network devices, and applications — providing real-time analysis and visualization.



### Container Run with `syslog` Log Driver:

To send container logs directly to the **syslog service** running on your host (or remote syslog server), you can start a container using the `syslog` logging driver.


```
docker run --name myapp -d --log-driver syslog nginx
```



### Configure Syslog to Send Logs to Graylog:
This configuration sends all syslog messages to Graylog running on UDP port `1514`.


_Edit `/etc/rsyslog.conf` or a new file under `/etc/rsyslog.d/graylog.conf`:_
```
*.* @127.0.0.1:1514;RSYSLOG_SyslogProtocol23Format
```


```
systemctl restart rsyslog
```



### Generate password and secret: 


_To Generate SHA-256 password:_
```
echo -n 'admin123' | sha256sum
```


_Install `pwgen` utility:_
```
apt install pwgen -y    # Debian/Ubuntu

dnf install pwgen -y    # Rocky Linux
```


_To Generate a random secret:_
```
pwgen -N 1 -s 96
```




### Create `docker-compose.yml` file:


Create a new file `docker-compose.yml`:
```
services:
  mongo:
    image: mongo:5.0
    container_name: mongo
    restart: always

  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.10.2
    container_name: es01
    environment:
      - discovery.type=single-node
      - ES_JAVA_OPTS=-Xms512m -Xmx512m
      - action.auto_create_index=true
    ulimits:
      memlock:
        soft: -1
        hard: -1
    restart: always

  graylog:
    image: graylog/graylog:5.2
    container_name: graylog01
    depends_on:
      - mongo
      - elasticsearch
    environment:
      - GRAYLOG_HTTP_EXTERNAL_URI=http://localhost:9000/
      - GRAYLOG_ROOT_PASSWORD_SHA2=<your_password>
      - GRAYLOG_PASSWORD_SECRET=<your_secret>
      - GRAYLOG_MONGODB_URI=mongodb://mongo:27017/graylog
      - GRAYLOG_ELASTICSEARCH_HOSTS=http://es01:9200
    ports:
      - "9000:9000"      # Web UI
      - "1514:1514/udp"  # Syslog UDP input
    restart: always

```


_Start all containers:_
```
docker compose up -d
```

### Access Dashboard: 
Visit in browser: `http://<your-server-ip>:9000`


#### Create a Syslog UDP Input:

1. Create a Syslog UDP Input: 
    - Navigate to `System` → `Inputs` → select `Syslog UDP` → `Launch new input`:
    - Title: node1_syslog
    - Bind address: 0.0.0.0
    - Port: 1514

2. Then Navigate to `Search` to view incoming logs. 



This configuration transforms your server into a complete Syslog management and visualization platform, making it easy to collect, analyze, and search logs across your entire infrastructure.
