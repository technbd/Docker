
## Docker Install Centos-7: 

To install Docker on Centos 7, you can follow these steps:


### Uninstall old versions:

Older versions of Docker went by docker or docker-engine. Uninstall any such older versions before attempting to install a new version, along with associated dependencies.

Images, containers, volumes, and networks stored in `/var/lib/docker/` aren't automatically removed when you uninstall Docker.

```
yum remove -y docker docker-client docker-client-latest docker-common docker-latest docker-latest-logrotate  docker-logrotate docker-engine
```


### Install the Dependencies:

```
yum update -y
yum install device-mapper-persistent-data lvm2 wget -y
```



### Set up the repository:
Before you install Docker Engine for the first time on a new host machine, you need to set up the Docker repository. Install the `yum-utils` package (which provides the `yum-config-manager` utility) and set up the repository.

```
yum install -y yum-utils
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
```

Or,

```
cd /etc/yum.repos.d/
wget https://download.docker.com/linux/centos/docker-ce.repo
```



_To install the latest version:_

```
yum install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

Install a specific version by its fully qualified package name, which is the package name (`docker-ce`) plus the version string (2nd column), separated by a hyphen (`-`). For example, `docker-ce-3:25.0.5-1.el7`.

Replace `<VERSION_STRING>` with the desired version and then run the following command to install: 


```
yum install -y docker-ce-<VERSION_STRING> docker-ce-cli-<VERSION_STRING> containerd.io docker-buildx-plugin docker-compose-plugin
```

### Verify Docker Container: 

```
docker search busybox
```


```
docker pull busybox
```


_Attached Mode:_
```
docker run -it busybox

or,

docker run -it busybox sh
```


_Detached Mode:_
```
docker run --name busybox1 -dit busybox
```





### Managing Docker as a Non-root User:
By default, Docker requires root privileges to run commands. However, granting root access to Docker can be a security risk. 

```
usermod -aG docker <user_name>
```

## Docker Autocompletion:

```
yum install bash-completion -y 

apt install bash-completion -y
```


_For Docker:_
```
curl https://raw.githubusercontent.com/docker/docker-ce/master/components/cli/contrib/completion/bash/docker -o /etc/bash_completion.d/docker.sh
```


```
ll /etc/bash_completion.d/docker.sh

-rw-r--r--. 1 root root 115K Oct  2 06:32 /etc/bash_completion.d/docker.sh
```


_For Docker-compose:_

```
curl -L https://raw.githubusercontent.com/docker/compose/1.25.1/contrib/completion/bash/docker-compose -o /etc/bash_completion.d/docker-compose
```


```
ll /etc/bash_completion.d/docker-compose
```
