
## Docker CLI Commands:

Here’s a list of commonly used Docker CLI commands categorized by different operations:



### Docker System Management:

_Show Docker disk usage:_
```
docker system df
```


_Remove unused data (containers, images, volumes, etc.):_
```
docker system prune
```


_Display system-wide information:_
```
docker system info
```



## Search for an Image:

```
docker search <image-name>

docker search nginx
```


```
docker search nginx --filter=stars=3
docker search nginx --filter=stars=20
docker search nginx --filter is-official=true
```


### Download a Docker Image:

```
docker pull <image-name>

docker pull nginx
```



### List Downloaded Images:

```
docker images
docker image ls

docker image ls -q
```


### Image Inspect:

```
docker inspect nginx

or,

docker inspect <IMAGE_ID>
```


### Remove an Image:

```
docker rmi <image-id>
```

Or,

```
docker image rm $(docker image ls -q)
```


_To delete all existing images:_
```
docker rmi $(docker images -q -a)
```

## Run a Container:

- Container doing single process consume
- Container ID 64 char long 
- `Ctrl + C`: Container process stopped & not running background
- `-d` (detach): option tells Docker to run the Container as a Service in the background.


_Here is common options for `docker run` :_

- `docker run --help` : For help 

- `-d` or  `--detach` : Run container in background and print container ID.
- `--name` : Assign a name to the container.
- `--cpus` : Limit the number of CPU cores the container can use.
- `--cpuset-cpus` : Specify which CPUs the container can use (e.g., 0, 0-2, 0,1).
- `-m` or `--memory` : Set the maximum memory limit (e.g., 512m, 1g).
- `--network` : Connect a container to a specific network.
- `-p` or `--publish` : Publish a container's port(s) to the host (e.g., 80:80).
- `--expose` : Expose a port without publishing it to the host.
- `--link` : Link to another container (deprecated).
- `-v` or `--volume` : Mount a volume (e.g., host_path:container_path).
- `-e` or `--env` : Set environment variables (e.g., MY_VAR=value).
- `--user` : Specify the username or UID and optionally group or GID to use when running the container.
- `CMD` : Command to run in the container (default CMD of the image).
- `--entrypoint` : Overwrite the default entry point of the image.
- `-t` or `--tty` : Allocate a pseudo-TTY.
- `-i` or `--interactive` : Keep STDIN open even if not attached.
- `-h` or `--hostname` : Set a hostname for the container.



```
docker run <image-name>

docker run nginx
```


_To run the container in detached mode (in the background):_
```
docker run -d nginx
```


_To map ports (mapped port 8080 on your host machine to port 80 inside the container.):_
```
docker run -d -p 8080:80 nginx

docker run --name web1 -d -p 8080:80 nginx
```

### Docker Resource Allocate:

```
docker stats
```

#### Limit CPU usage:

- `--cpus` : Specifies the number of CPU cores the container can use. For example, `--cpus="2"` limits the container to use 2 CPU cores.
- `--cpuset-cpus=0` : When you specify `--cpuset-cpus=0`, it means the **container is restricted to run only on CPU core 0**. The container cannot use any other CPU cores. The container will only be able to execute processes on CPU core 0, even if the host machine has multiple cores available.

- `--cpuset-cpus="0,1"` : Container can run on both CPU 0 and CPU 1.
- `--cpuset-cpus="0-2"` : Container can run on CPU cores 0, 1 and 2.


```
docker run --name web3 -dit --cpus="2" -p 8082:80 nginx
```


```
docker run --name web3 -dit --cpuset-cpus=0 -p 8082:80 nginx
```


```
docker run --name web3 -dit --cpuset-cpus=0-2 -p 8082:80 nginx
```




#### Limit memory usage:

- `--memory` : Specifies the maximum amount of memory the container can use. You can define the value in bytes (e.g., 128m, 1g).

```
docker run --name web4 -dit --memory=20m -p 8083:80 nginx
```





### List Running Containers:
- `-a`, `--all` : Show all containers (both running and stopped).
- `-l`, `--latest` : Show only the most recently created container (either running or stopped).
- `-s`, `--size` : Show the total file size of each container, including the file system and any data volumes.
- `-q`, `--quiet`: Display only container IDs, useful for scripting or batch operations.
- `-f`, `--filter` : Filter the containers by specific conditions (can use multiple --filter flags).
- `--format` : Pretty-print the output using a Go template.


```
docker ps
docker ps -a
docker ps -l
```


```
docker ps -f "name=web1"
docker ps -f "status=exited"
```


```
docker ps --format "table {{.Names}}\t{{.Status}}"
docker ps --format "table {{.Names}}\t{{.Image}}"
docker ps --format "table {{.Names}}\t{{.Command}}"
docker ps --format "table {{.Names}}\t{{.Ports}}"
```

### Container Logs:

Here are a few basic Docker commands to help you get started with Docker logs and metrics:

- Show container logs: `docker logs container_Name`
- Show only new logs: `docker logs -f container_Name`
- Show CPU and memory usage: `docker stats`
- Show CPU and memory usage for specific containers: `docker stats` `container_Name1` `container_Name2`
- Show running processes in a container: `docker top container_Name`
- Show Docker events: `docker events`
- Show storage usage: `docker system df`


```
docker logs <container_name>

docker logs <container_id>
```


```
docker logs -f <container_name>

docker logs -f <container_id>
```

## Execute a Command (`exec`) in a Running Container:

```
docker exec -it <container_name> <command>

docker exec -it <container_id> <command>
```


```
docker exec -it web1 cat /etc/shells
```


```
docker exec -it web1 df -h
```


```
docker exec -it web1 ls /usr/share/nginx/html

50x.html  index.html
```


```
docker exec -it web1 cat /usr/share/nginx/html/index.html

Hello NGINX
```



_To open a Bash shell inside a running container:_
```
docker exec -it web1 bash
```


```
cd /usr/share/nginx/html

echo "Hello NGINX" > index.html
```

## Docker `cp` Command:


### Copy Host to Container:

_Syntax:_
```
docker cp <source_path> <container_name or container_name>:<destination_path>
```


```
mkdir /home/idea/technbd
echo "Hello Technbd" > /home/idea/technbd/index.html
```


```
### Directory Copy:
docker cp /home/idea/technbd web1:/usr/share/nginx/html


### File copy:
docker cp /home/idea/technbd/index.html web1:/usr/share/nginx/html
```


```
docker exec -it web1 ls /usr/share/nginx/html

50x.html  index.html  technbd
```


### Copy File From Container to Host:

_Syntax:_
```
docker cp <container_name or container_id>:/path/in/container/file.txt  /path/on/host/
```


_File copy:_
```
docker cp web1:/etc/passwd /home/idea/
```


_Directory copy:_
```
docker cp web1:/usr/share/nginx/html /home/idea/
```

## Docker `commit` command: 

The docker `commit` command is used to create a new Docker image from the changes made to a running or stopped container.

- `-m "message"` : Add a commit message to describe the changes made in the new image.
- `-a "author"` : Specify the author of the image (e.g., your name).


_Syntax:_
```
docker commit <container_id> <new_image_name>

docker commit [OPTIONS] <container_id> <new_image_name>
```

## Docker Image Save and Load:

- `-o`` output_file_name.tar`: Specifies the output file.
- `-i` , `--input` : This option tells Docker to load the image from the specified tar file.



#### Image Save:

_Syntax:_
```
docker save -o <path/of/output_file_name.tar> <image_name>
docker save -o <path/of/output_file_name.tar> <image_name:version>
```


```
docker save -o /home/idea/web2-new-v1.tar  web2-new:v1
```


```
ll /home/idea/web2-new-v1.tar

-rw------- 1 idea idea 191878144 Oct  3 01:51 /home/idea/web2-new-v1.tar
```


#### Image Load:

_Syntax:_
```
docker load -i <input_file_name.tar>
```


```
docker load -i /home/idea/web2-new-v1.tar
```


```
docker images
```
