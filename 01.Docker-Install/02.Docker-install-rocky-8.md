## Docker Install Rocky-8: 

There are two editions of Docker: the **community edition (Docker CE)** and the **enterprise edition (Docker EE)**. Docker EE is an enterprise-ready container management solution.

To install Docker on Rocky Linux 8, you can follow these steps:


### Install Docker:

To install following packages: 

- `docker-ce` : This package provides the underlying technology for building and running docker containers (dockerd) 
- `docker-ce-cli` : Provides the command line interface (CLI) client docker tool (docker)
- `containerd.io` : Provides the container runtime (runc)
- `docker-compose-plugin` : A plugin that provides the 'docker compose' subcommand



_Install necessary packages:_

```
dnf install -y dnf-utils
yum install -y yum-utils
```


_Use the `dnf` utility to **add the Docker repository** to your Rocky Linux server:_

```
dnf config-manager --add-repo https://download.docker.com/linux/rhel/docker-ce.repo
```


_Install Docker & Additional Packages:_

```
dnf install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
```



### Uninstall Docker:

```
dnf remove -y docker-ce docker-ce-cli containerd.io
```


```
rm -rf /var/lib/docker
rm -rf /var/lib/containerd
```




### Load br_netfilter Module:

_First, check the current values of the settings:_

```
sysctl net.bridge.bridge-nf-call-iptables
sysctl net.bridge.bridge-nf-call-ip6tables
```


_Load the `br_netfilter` modules into the running Linux kernel:_

```
modprobe br_netfilter

modinfo br_netfilter
```



_Add this two lines:_
```
vim /etc/sysctl.conf

net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1
```


_Reload `sysctl` Settings:_
```
sysctl -p

sysctl --system
```

### Verify Docker Container: 

```
docker search busybox
```


```
docker pull busybox
```


_Attached Mode:_
```
docker run -it busybox

or,

docker run -it busybox sh
```


_Detached Mode:_
```
docker run --name busybox1 -dit busybox
```





#### Optionally allow a non-root user to manage docker:
Add a non-root user to the `docker` group to allow the user to manage docker without `sudo`.

```
usermod -a -G docker <user_name>

or,

sudo usermod -aG docker $USER
sudo usermod -aG docker $(whoami)
```


```
id $USER
```

## Docker Autocompletion:

```
yum install bash-completion -y 

apt install bash-completion -y
```


_For Docker: Worked v27.3.1:_
```
curl https://raw.githubusercontent.com/docker/docker-ce/master/components/cli/contrib/completion/bash/docker -o /etc/bash_completion.d/docker.sh
```


```
ll /etc/bash_completion.d/docker.sh

-rw-r--r--. 1 root root 115K Oct  2 06:32 /etc/bash_completion.d/docker.sh
```



_For Docker-compose:_

```
curl -L https://raw.githubusercontent.com/docker/compose/1.25.1/contrib/completion/bash/docker-compose -o /etc/bash_completion.d/docker-compose

or,

curl -L https://raw.githubusercontent.com/docker/compose/$(docker-compose version --short)/contrib/completion/bash/docker-compose -o /etc/bash_completion.d/docker-compose
```




```
ll /etc/bash_completion.d/docker-compose
```
