
## Docker Portainer:

Portainer is a management UI designed for Docker environments, helping you manage and visualize Docker containers, images, networks, and volumes. It's lightweight and provides a web interface to interact with your Docker setup, making container management much easier, especially if you prefer visual tools over the command line. Portainer works for both standalone Docker environments and Docker Swarm clusters. By default, Portainer Server will expose the UI over port `9443` and expose a TCP tunnel server over port `8000`.



### 1. Portainer on `docker run`: 


```
docker pull portainer/portainer-ce
```


```
docker volume create portainer_data
```


```
docker run --name portainer1 -dit -p 8000:8000 -p 9443:9443 --restart=always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce
```



```
docker restart portainer1
```



#### Web Console:

Now, that the installation is complete, you can log into your Portainer Server instance by opening a web browser and going to: `https://<your-server-ip>:9443`







### 2. Portainer on `docker-compose.yaml`


```
services:
  portainer:
    image: portainer/portainer-ce:latest
    container_name: portainer1
    ports:
      - "8000:8000"
      - "9443:9443"
    restart: always
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
      - portainer_data:/data

volumes:
  portainer_data:

```


```
docker-compose config
```


```
docker-compose up
```



#### Portainer Web Interface:
After starting Portainer, you can access the UI at `https://<your-server-ip>:9443`
