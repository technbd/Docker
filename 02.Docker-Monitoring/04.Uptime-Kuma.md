
## Uptime Kuma - Monitoring Tool:

Uptime Kuma is an easy-to-use self-hosted monitoring tool. Uptime Kuma can monitor a wide range of targets to help you keep track of the status of your websites, services, and network resources. 

Here are some of the most common types of monitoring it supports:

- HTTP(s) Monitoring
- TCP Port Monitoring
- Ping (ICMP) Monitoring
- HTTP(s) Keyword Monitoring
- DNS Monitoring
- Docker Container Monitoring
- TLS Certificate Expiry Monitoring
- Push Monitoring



### Run Uptime Kuma with Docker: 

```
docker run --name uptime-kuma -d --restart=always -p 3001:3001 -v ./uptime-kuma:/app/data louislam/uptime-kuma:1
```


Or,


```
#docker-compose.yaml

services:
  uptime-kuma:
    image: louislam/uptime-kuma:1
    container_name: uptime-kuma
    restart: always

    ports:
      - "3001:3001"
      
    volumes:
      - ./uptime-kuma:/app/data
      - /var/run/docker.sock:/var/run/docker.sock
```



```
docker-compose config
```


```
docker-compose up -d
```


### Uptime Kuma Web Console:

Once the service is up, you can access Uptime Kuma at: `http://<your_server_ip>:3001`
